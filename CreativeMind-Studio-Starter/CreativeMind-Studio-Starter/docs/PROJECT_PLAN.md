# Project Plan

## Product Goal
Build a working AI creative strategy room that validates, improves, and converts creative ideas into execution-ready plans.

## Must Have
- Raw idea input
- Multi-agent boardroom
- IBM Granite integration
- Creative scorecard
- Audience simulation
- Ethical audit
- Refined idea
- Roadmap
- Final report

## Should Have
- Pivot slider
- Trend Radar
- Cross-platform adaptation
- Idea version history

## Stretch
- Virality Twin
- Multiplayer war room
- Creator DNA
- Post-launch feedback loop

## Phases
### Phase 0 — Repository
Create repo, branches, docs, schemas, naming rules.

### Phase 1 — Static Frontend
Landing, idea form, boardroom UI, scorecard, audience cards, refined idea, roadmap.

### Phase 2 — Backend Core
FastAPI, project creation, session storage, schemas, database.

### Phase 3 — IBM Granite
Granite service, three agents, two-turn debate, synthesis, JSON validation.

### Phase 4 — Integration
Connect frontend and backend, loading/error states, render all outputs.

### Phase 5 — Differentiation
Ethical Auditor, audience simulation, pivot, Trend Radar, platform adaptation.

### Phase 6 — Submission
Testing, responsiveness, deployment, README screenshots, demo video, final submission.

## First Integrated Milestone
- User submits a raw idea
- Backend stores project
- IBM Granite runs three agents
- Agents use earlier context
- Synthesis returns validated JSON
- Frontend renders debate, scores, refined idea, roadmap

## Definition of Done
A feature is done only when it works locally, is tested, documented, reviewed, and merged into `develop`.
